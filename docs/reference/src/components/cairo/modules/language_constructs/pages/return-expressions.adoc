= Return expressions

A return expression exits the enclosing function early and specifies
the value to return to the caller. Return expressions use the `return`
keyword followed by an optional expression.

== Syntax

[source,cairo]
----
return;           // Returns unit type ()
return <expr>;    // Returns the value of <expr>
----

== Basic usage

Return expressions exit the function immediately, skipping any
remaining code in the function body.

[source,cairo]
----
fn check_positive(x: i32) -> felt252 {
    if x < 0 {
        return 0;  // Exit early if negative
    }
    x.into()
}
----

== Return type

The type of a return expression is the xref:never-type.adoc[never type]
(`!`), which indicates the expression does not produce a value but
instead changes control flow. This allows return expressions to be
used in any context regardless of the expected type.

[source,cairo]
----
let x: u32 = {
    if condition {
        return 42;  // Type is never (!), coerced to u32
    }
    10
};
----

== Implicit returns

Cairo functions can return values without using `return` by using a
tail expression (an expression without a semicolon at the end of the
function body).

[source,cairo]
----
fn add(a: u32, b: u32) -> u32 {
    a + b  // Implicit return (no semicolon)
}

fn add_explicit(a: u32, b: u32) -> u32 {
    return a + b;  // Explicit return (equivalent)
}
----

Both forms are equivalent, but tail expressions are more idiomatic in
Cairo.

== Returning unit type

When a return expression has no value (`return;`), it returns the
xref:unit-type.adoc[unit type] `()`.

[source,cairo]
----
fn maybe_return(condition: bool) {
    if condition {
        return;  // Returns (), exits early
    }
    // Rest of function...
}
----

This is equivalent to `return ();`.
