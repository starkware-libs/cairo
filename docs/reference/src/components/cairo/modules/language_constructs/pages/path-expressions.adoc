= Path expressions

A path expression is a standalone xref:path.adoc[concrete path] used as an
expression.
It resolves to one of these items:

* A local variable
* A xref:constant-items.adoc[constant item]
* A generic const parameter (currently unsupported)

== Local Variables

The simplest path expression is a single identifier referring to a local
variable:

[source,cairo]
----
const fn unwrap_or<+Destruct<T>>(self: Option<T>, default: T) -> T {
    match self {
        Some(x) => x,
        None => default,
    }
}
----

Here `x` and `default` are path expressions that evaluate to local variables.

Variables bound in patterns are also local variables. In the example above,
`x` is bound by the pattern `Some(x)`.

== Constants

A path expression can refer to a constant item:

[source,cairo]
----
const TRUE: bool = true;
const IF_CONST_TRUE: felt252 = if TRUE {
    4
} else {
    5
};
----

The identifier `TRUE` is a path expression that evaluates to the constant
value.
